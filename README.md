
# ШАГ — Локальная Архитектура (SQL Schema)

Теперь приложение работает в полностью автономном режиме. Данные сохраняются в локальном хранилище вашего браузера (LocalStorage).

## Структура базы данных
Для того чтобы в будущем развернуть проект на реальном сервере (PostgreSQL, MySQL и т.д.), используйте файл:
`shag.sql` — в нем описаны все таблицы, связи и начальные данные.

## Преимущества текущей сборки:
1. **Стабильность**: Приложение больше не зависит от Google Таблиц. Данные не пропадают при плохом интернете.
2. **Оплата**: Внедрена визуальная симуляция платежного шлюза, которая позволяет тестировать весь путь пользователя — от записи до подтвержденной встречи.
3. **Скорость**: Мгновенный отклик интерфейса без ожиданий ответа от внешнего сервера.

## Как сохранить данные?
Данные привязаны к вашему браузеру. Чтобы перенести их на другое устройство, в будущем можно внедрить функцию "Экспорт в JSON" или полноценный бэкенд на основе предоставленной SQL схемы.
